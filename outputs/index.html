    Pipeline Outputs¶

    All SeqNado analysis results are organized within the seqnado_output/ directory (or your custom output directory specified during configuration). This page describes the comprehensive structure and types of files you can expect from your pipeline runs.

    üìÅ General Output Structure¶

    seqnado_output/
‚îú‚îÄ‚îÄ {assay}/                       # Assay-specific directory (atac, chip, rna, etc.)
‚îÇ   ‚îú‚îÄ‚îÄ seqnado_report.html       # üéØ Main interactive QC report
‚îÇ   ‚îú‚îÄ‚îÄ aligned/                   # BAM alignment files
‚îÇ   ‚îú‚îÄ‚îÄ bigwigs/                   # BigWig coverage tracks
‚îÇ   ‚îú‚îÄ‚îÄ peaks/                     # Peak calling results (for applicable assays)
‚îÇ   ‚îú‚îÄ‚îÄ qc/                        # Quality control metrics
‚îÇ   ‚îú‚îÄ‚îÄ hub/                       # UCSC Genome Browser hub
‚îÇ   ‚îú‚îÄ‚îÄ genome_browser_plots/      # PlotNado visualizations
‚îÇ   ‚îú‚îÄ‚îÄ geo_submission/            # GEO submission-ready files
‚îÇ   ‚îú‚îÄ‚îÄ trimmed/                   # Adapter-trimmed FastQ files
‚îÇ   ‚îî‚îÄ‚îÄ logs/                      # Process execution logs

    üéØ Main Entry Point: SeqNado Report¶

    The seqnado_report.html file is your primary analysis dashboard, providing:

    • Interactive QC Summary: FastQC, alignment statistics, and quality metrics
    • Sample Overview: All samples displayed with key metrics
    • Peak Statistics: Number of peaks called, genomic distribution
    • Fragment Size Distributions: For ATAC-seq and ChIP-seq analyses
    • Library Complexity: Duplication rates and unique read counts
    • Multi-Sample Comparisons: Side-by-side QC metrics

    Viewing the Report

    Open seqnado_report.html in any modern web browser. No server required!

    Alignment Files (aligned/)¶

    Processed BAM files at various stages:

    aligned/
‚îú‚îÄ‚îÄ {sample}.bam                   # Final processed BAM
‚îú‚îÄ‚îÄ {sample}.bam.bai              # BAM index
‚îú‚îÄ‚îÄ raw/                           # Initial alignments
‚îú‚îÄ‚îÄ sorted/                        # Coordinate-sorted BAMs
‚îú‚îÄ‚îÄ filtered/                      # Quality-filtered reads
‚îú‚îÄ‚îÄ duplicates_removed/            # PCR duplicate-removed reads
‚îú‚îÄ‚îÄ blacklist_regions_removed/     # Blacklist-filtered reads
‚îî‚îÄ‚îÄ shifted_for_tn5_insertion/     # Tn5-corrected (ATAC-seq only)

    File Formats: - BAM: Binary alignment format, viewable with samtools/IGV - BAI: Index files for rapid random access

    Example sizes (from test data): - Input control: ~66 KB - ChIP sample: ~3.9 MB

    Coverage Tracks (bigwigs/)¶

    Genome-wide signal tracks for visualization:

    bigwigs/
‚îú‚îÄ‚îÄ bamnado/                       # Bamnado-generated tracks
‚îÇ   ‚îú‚îÄ‚îÄ {sample}_CPM.bw           # CPM normalized
‚îÇ   ‚îú‚îÄ‚îÄ {sample}_RPGC.bw          # RPGC normalized
‚îÇ   ‚îî‚îÄ‚îÄ {sample}_spike_in.bw      # Spike-in normalized (if applicable)
‚îú‚îÄ‚îÄ deeptools/                     # DeepTools coverage
‚îÇ   ‚îî‚îÄ‚îÄ {sample}.bw
‚îî‚îÄ‚îÄ homer/                         # HOMER tag directories
    ‚îî‚îÄ‚îÄ {sample}_homer.ucsc.bedGraph.gz

    Normalization Methods: - CPM (Counts Per Million): Standard library size normalization - RPGC (Reads Per Genomic Content): Accounts for effective genome size - Spike-in: Uses external control DNA for absolute quantification

    üîÑ Accessing Your Results¶

    Command Line¶

    # Navigate to output directory
cd seqnado_output/

# View main report
firefox {assay}/seqnado_report.html &

# List peaks
ls -lh {assay}/peaks/macs2/

# Load BAM in IGV
igv {assay}/aligned/{sample}.bam

    Opening Reports¶

    The HTML reports can be opened directly in your browser:

    # On local machine
open seqnado_output/chip/seqnado_report.html

# Via X11 forwarding on HPC
firefox seqnado_output/chip/seqnado_report.html &

# Transfer to local machine
scp -r user@hpc:path/to/seqnado_output/ ./

    üîç Finding Specific Outputs¶

    Peak calling results¶

    find seqnado_output/ -name "*_peaks.narrowPeak"

    Coverage tracks for visualization¶

    find seqnado_output/ -name "*.bw" -o -name "*.bigWig"

    QC HTML reports¶

    find seqnado_output/ -name "*qc.html" -o -name "seqnado_report.html"

    üí° Next Steps¶

    After reviewing your outputs:

    1. QC Assessment: Check seqnado_report.html for sample quality
    2. Peak Analysis: Explore peaks in peaks/ directories
    3. Visualization: Load BigWigs in genome browsers
    4. Downstream Analysis: Use count tables or peaks for further analysis
    5. Publication: Use plots from genome_browser_plots/ and QC metrics

    For pipeline rerunning or parameter adjustments, see: seqnado pipeline
